# I Can See!

A colorblind-friendly color identifier for iPhone. Aim your camera at
anything — or load a photo — and I Can See! names the color in big,
readable text.

Built for the red/green and blue/purple confusions that affect ~99% of
people with color vision deficiency.

## Features

- **Live camera mode** — a crosshair samples the centre of the frame in
  real time and names the colour. Tap to freeze/unfreeze.
- **Photo mode** — load any image from Photos or Files, pinch to zoom,
  and drag a dropper to identify a specific pixel.
- **31-name palette** curated for clarity over precision, matched in CIE
  Lab space so dark navy reads as *Navy* (not *Black*).
- **Color correction** — if a reading seems wrong, tap to manually pick the correct color. Corrections are tracked anonymously via TelemetryDeck to improve matching.
- **Local-only** — camera frames are processed on-device, no uploads,
  no account.

## Build

```sh
brew install xcodegen           # one-time
./scripts/generate.sh           # generates ICanSee.xcodeproj
open ICanSee.xcodeproj
```

Requires Xcode 15+ and iOS 17.0 deployment target.

The app icon PNG is regenerated by `scripts/generate-app-icon.swift` and
checked into `ICanSee/Assets.xcassets/AppIcon.appiconset/`.

## Layout

- `ICanSee/Models/` — `NamedColor`, `ColorMatcher`
- `ICanSee/Services/` — `CameraService`, `ImageSampler`, `Analytics`
- `ICanSee/Views/` — `RootView`, `CameraPreviewView`, `PhotoInspectorView`,
  `ColorReadoutView`, `FirstRunGuideView`
- `ICanSee/Localizable.xcstrings`, `ICanSee/InfoPlist.xcstrings` —
  region-specific spellings (*color* vs *colour*)
- `docs/` — App Store submission metadata and the marketing/support/privacy
  pages required by App Store Connect

## Privacy

See [docs/privacy.html](docs/privacy.html). Short version: nothing leaves
your phone except anonymous TelemetryDeck usage signals (`App.launch`,
`Camera.authorized`/`Camera.denied`, and correction events such as
`Reading.corrected`/`Reading.correctionCleared`).
